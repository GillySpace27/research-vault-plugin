# Research vault

A personal research-task vault — one markdown file per project, GitHub-style
checkbox tasks with Obsidian Tasks emoji metadata, a daily journal, and a
two-tier memory of people, projects, and shorthand. Driven by the
**research-vault** Claude Code plugin.

## Layout

- `CLAUDE.md` — hot-cache memory (roster, codenames, vault path) + plugin pointer.
- `projects.md` — the project manifest (one line per project file).
- `<project>.md` — one file per project; tasks live under `## Open`.
- `inbox.md` — capture catch-all; route during `/triage`.
- `proposal-solicitations.md` / `proposal-ideas.md` — funding-call tracking.
- `daily_notes/YYYY-MM-DD.md` — the daily journal.
- `memory/` — deep storage: `glossary.md`, `people/`, `projects/`, `context/`.
- `dashboard.md` — Obsidian Tasks queries (run `/dashboard` for an HTML board).

## Day to day

`/capture` a task · `/update` to reconcile · `/status` for the big picture ·
`/journal` to log · `/triage` weekly · or just talk to Claude.

See the plugin's GETTING-STARTED for the full walkthrough.
